C存储类别笔记

发表于 2020-05-12 更新于 2020-05-15

存储类别(Storage class)

A storage class represents the visibility and location of a variable.

参考

  • Storage Classes in C
  • Storage-class specifiers

存储类别描述以下几个方面：

  • 变量作用域
  • 变量存储的location
  • 变量的初始化值
  • 变量的生命周期（lifetime）
  • 谁能访问变量

C语言把被存储的每个值占用一定的物理内存块叫做对象(object)

作用域

  • 块作用域（block scope）
  • 函数作用域（function scope）
  • 函数原型作用域（function prototype scope）
  • 文件作用域（file scope）

翻译单元（translation unit）

C预处理实际上用包含的头文件替换#include指令，进而编译器源代码文件和所有的头文件都看成是一个包含信息的单独文件，即翻译单元

链接

三种链接属性：

  • 外部链接
    可以在多文件程序中使用
  • 内部链接
    只能在一个翻译单元中使用，可以用关键字static声明变量
  • 无链接
    块作用域、函数作用域或函数原型作用域的变量为无链接变量

参考文章what-is-external-linkage-and-internal-linkage

存储期

存储期描述标识符访问的对象的生存期

四种存储期

  • 静态存储期
    • 对象在程序执行期间一直存在
    • 文件作用域变量具有静态存储期（可以是内部链接、外部链接或无链接）
  • 线程存储期
    用于并发设计，对象从被声明到线程结束一直存在。以关键字_Thread_local声明一个对象时，每个线程都获得该变量的私有备份
  • 自动存储期
    块作用域通常具有自动存储期。自动变量占用的内存可视为一个可重复使用的工作区或暂存区
  • 动态分配存储期

五种存储类别

自动变量（auto）

属于自动存储类别的变量具有自动存储期、块作用域且无链接，可以现实使用关键字auto

如果未初始化自动变量，它的值是为定义的

寄存器变量（register）

自动存储期、块作用域且无链接，无法获取寄存器地址

register类别相比直接命令更像是一种请求，编译器必须根据寄存器或最快可用内存的数量来
衡量请求，或者忽略请求，此情况下寄存器变量就成了普通变量

静态变量

静态的意思是该变量在内存中原地不动，并不是值不变

变量在块作用域中

具有静态存储期、块作用域和无链接，在块中以static声明变量，也可用局部静态变量来描述

变量为外部链接

具有静态存储期、文件作用域和外部链接，该类别有时称为外部存储类别（external storage
class），属于该类别的变量称为外部变量（external variable）

初始化外部变量

  • 未初始化的外部变量，被自动初始化为0
  • 只能使用常量表达式初始化文件作用域变量

定义和声明

第1次声明被称为定义式声明，第2次被称为引用式声明
外部变量只能初始化一次，且必须在定义该变量时进行

存储类别选择

保护性程序设计的黄金原则：“按需知道”。尽量在函数内部解决函数的任务，只共享些需要共享的变量

ANSI C类型限定符

const限定符

A keyword applied to a data type that indicates that the data is read only.

In the C family of languages the const is part of the type, not part of object. For example, int const x = 1, means “(int const) x”

  • 在指针和形参声明中使用const
    const放在左侧的任意位置，限定了指针指向的数据不能改变；放在的右侧，限定了指针本省不能改变
    如果一个指针仅用于给函数访问值，应该将其声明为一个指向const限定类型的指针
  • 对全局数据使用const
    文件间共享数据策略
    • 遵循外部变量的常用规则——在一个文件中使用定义式声明，在其他文件中使用引用式声明（用extern）
    • 把const变量放在一个头文件中，然后在其他文件中包含该头文件

volatile限定符

告诉编译器，代理（不是变量所在的程序）程序可以改变该值

用法和const一样，不过两者可以结合起来使用，主要用于编译器优化

参考：
Understanding “volatile” qualifier in C | Set 1 (Introduction)
Understanding “volatile” qualifier in C | Set 2 (Examples)

restrict限定符

目的

允许编译器优化某部分代码以更好地支持计算，只能用于指针（包含函数的形参），表明指针是访问数据对象唯一且初始的方式

两个目标读者

  • 编译器，告诉编译器可以自由假定一些优化方案
  • 用户，告诉程序员要使用满足restrict要求的参数

参考
restrict wiki
