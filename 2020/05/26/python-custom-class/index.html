发表于 2020-05-26 更新于 2020-05-28

classmethod与staticmethod

classmethod

NOTES

  • 调用方式1是在实例上调用类方法，类作为参数隐式的传给cls
  • 调用方式2即约定俗成的调用方式，在类上调用类方法

staticmethod

staticmethod表现就如同常规方法一样，不用点在于，既可以在实例上调用也可以在类上调用。静态方法有点像C中具有块作用域的静态变量，一旦模块被导入，类中的静态方法就会在内存栈中开辟一块区域，运行期间一直不变，因此实例上调用和类上调用的内存地址是一样的

作用

这两个方法从语法上较简单，作用要从面对对象的设计角度来考量：

  • 类方法的作用是实现其他语言如Java中构造方法重载的功能
  • 静态方法主要用来设计类对象的工具方法

参考:

  • Difference between staticmethod and classmethod
  • Python’s Instance, Class, and Static Methods Demystified

private和protected属性

Python中对类的属性访问限制没有语法上限制，只有约定俗成的用法：

  • 两个前导下划线表示私有属性
  • 一个前导下划线表示受保护的属性

实际运用中一般只使用一个前导下划线，两个前导下划线一般是Python库中使用的

slots类属性

__slots__ allows us to explicitly declare data members (like properties) and deny the creation of __dict__ and __weakref__(unless explicitly declared in __slots__ or avaliable in a parent)

Python的各个实例属性存储在dict属性中

可以动态的添加属性到实例的__dict__属性中，而__slot__阻止了这一行为，__slots__告诉解释器实例中就包含里面定义的属性，因而在同时创建大量的实例情况下会显著的减少内存的使用

NOTES：

  • 实例只能拥有__slots__中列出的属性，无法给实例变量赋值新的属性
  • __slots__是在类级别为每个变量创建描述符来实现的
  • 除非子类也定义__slots__属性，否则解释器会忽略继承的__slots__属性
