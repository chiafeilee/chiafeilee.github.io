Python对象引用、可变性笔记

发表于 2020-05-14 更新于 2020-05-21

==运算符比较两个对象的值（对象中保存的值），is比较对象的标识（内存地址）

相等性

==运算符比较两个对象的值（对象中保存的值），is比较对象的标识（内存地址）
在变量和单例值(singleton)之间的比较时，应该使用is，例如None

为什么用is比较None

什么是None？

The sole value of the type NoneType. None is frequently used to represent the absence of a value, as when default arguments are not passed to a function

Facts

  • ==运算符是语法糖，等同于对象的x.__eq__(y)——继承自object对象，这个方法可被override
  • 由上一条得is运算符比==运算符要快，因为不用寻找特殊方法__eq__()，is直接比较ID
  • 关键字None是一个对象
  • None是一个singleton值——NoneType类型的单例值

  • Null in Python
  • The None Object
  • What is the None keyword in Python?

元组的相对不可变性

对象复制

Python中赋值表达式不复制对象，仅仅是创建一个链接到目标对象，即共享源对象的引用

浅复制(Shallow Copy)

复制最外层的容器，副本中元素(child objects)是源容器中元素的引用，构造方法及[:]或copy包中的copy方法做浅复制

深复制(Deep Copy)

副本不共享内部对象引用

浅复制和深复制的不同点仅仅在于复合对象(compund objects)上——对象包含其他对象，由此可得出一个简单的结论，即浅复制是一个one level deep

参考：
Shallow vs Deep Copying of Python Objects
Shallow and deep copy operations

函数调用

概念：块(block)

A block is a piece of Python program text that is executed as a unit —— Execution model

块有以下几种：

  • 模块(module)
  • 函数体(function body)
  • 类定义(class definition)
  • 交互式命令行
  • 脚本命令（解释器上使用-c选项指定命令）
  • 内置函数exec()和eval()执行的参数

概念：名称绑定(Name Binding)

Python中的“变量”和其他的语言如C、Go中的概念很不一样——先在内存中申请个“盒子”（定义变量），然后往里装东西（赋值），Python是先在内存里未命名的“盒子”里装东西，然后贴上“标签”（变量名），Python中“变量”通常称为“名称”（names）

Names refer to objects. Names are introduced by name binding operations —— Execution Model

名称绑定有6种情形：

  • 传给函数的形式参数
  • import语句
  • 类与函数定义
  • 以标识符为目标的赋值，如for循环的开头、with语句、except子句的as之后
  • import语句from...import *形式，绑定导入模块中定义的所有名称，私有变量除外（以下划线开头）
  • del语句

3种变量：

  • local variable
  • global variable
  • free variable

Free variables are not resolved in the nearest enclosing namespace, but int the global namespace

参考：

  • Binding of names

概念：作用域

A scope defines the visibility of a name within a block, is a textual region of a Python program where a namespace is direct accessible.

作用域定义了变量的可访问性和存储期（即生存期），通过把names映射到objects的字典来实现

概念：Namespace

A namespace is a container where names are mapped to objects, they are used to avoid confusions in cases where same names exist in different namesapces

namespace本质上就是变量存储的地方，通过属性__dict__可以访问到存储的names到objects的映射，不同的存储空间存储不同的名称，如下图：

图片来源：https://www.programiz.com/python-programming/namespace

因此对于namespace最重要的一点就是隔离了名称，分两部分看这个概念name+space，name指唯一标识符，space指标识符关联的scope

3种namespace说明：

  • Built-in Namespace
    Python解释器启动的时候创建，在运行时永久不会被删除
  • Global Namespace
    加载模块后，将创建全局namespace
  • Local Namespace
    函数调用时创建

  • Namespaces and Scope in Python

LEGB规则

name解析规则即LEGB，以名称作用域命名，分别是Local,Enclosing,Global,Built-in，并以此顺序解析名称


图片来源：https://www.askpython.com/python/python-namespace-variable-scope-resolution-legb

Local Scope

范围：函数或lambda表达式，在调用时创建

Enclosing Scope

范围：存在嵌套函数(nested function)

Global Scope

范围：Python程序、脚本或模块的最外层
Notes

  1. 如果需要在函数中修改全局名称的值，如果不使用global关键字，那么函数中的名称的修改只作用于函数本身，全局名称的值不会被修改
  1. 如果在函数中使用全局名称前既没有使用global关键字也没有绑定值，那么出抛出UnboundLocalError错误

范围：Python内置名称

运行Python解释器的时自动加载内置对象，交互控制台通过dir函数可查看

求值策略

求值策略决定两件事

when to evaluate the arguments of the function call and what kind of value to pass to the function
—— Evaluation strategy

最常见的求值策略就是call by value及call by reference，Python中用的是call by sharing策略，也叫call by object reference策略，以Python文档为准取第二种叫法。与前两个求值策略的区别，参考wiki_call_by_sharing

In particular it is not call by value because mutations of arguments performed by the called routine will be visible to the caller. And it is not call by reference because access is not given to the variables of the caller, but merely to certain objects.

我的看法是Python是动态类型的语言，所有都是对象，没有C、Go中的指针、引用，传给形参的都是对象，实际上表现的行为是call by value和call by reference的结合

参数是可变及不可变类型时的函数调用

参数是不可变类型时，如numbers、strings、tuples

参数是可变类型时，如list、dict、set

可变类型作为参数时调用的特殊情况

  • 可变类型作为默认参数

默认值在定义函数时计算，加载模块的时候也就成了函数对象的一个属性

  • 函数内部修改变量值

由作用域、Namespace等概念可知，在函数内部给变量绑定一个值，其实是创建一个本地变量、函数作用域的名称，不会影响外部的实参的值
